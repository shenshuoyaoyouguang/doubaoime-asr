# 上线部署与回滚预案（阶段版）

> 日期：2026-04-08  
> 状态：阶段性草案，待最终交付前补齐执行细节

## 1. 目标

在不改变核心业务行为的前提下，安全发布当前重构版本，并在出现兼容性或稳定性问题时可快速回退。

## 2. 适用范围

适用于：

- Windows 桌面代理发布
- overlay/native 组件随版本一起更新
- Python 运行环境或打包产物替换

## 3. 发布前检查

### 3.1 代码与验证
- 全量回归通过
- 定向回归通过
- 本轮变更已纳入阶段文档

### 3.2 运行时依赖
- Python 环境可用
- 以下 DLL 可正常加载：
  - `opus.dll`
  - `libgcc_s_seh-1.dll`
  - `libwinpthread-1.dll`

### 3.3 配置与数据
- 备份 `%APPDATA%/DoubaoVoiceInput/`
  - `credentials.json`
  - `logs/`
  - 其他运行时配置文件

### 3.4 构建产物
- overlay/native 构建完成
- agent 可正常启动

## 4. 建议发布步骤

### 4.1 灰度发布
1. 在小范围机器安装/替换新版本
2. 验证以下场景：
   - 热键触发
   - worker 启动与结束
   - interim / final 显示
   - 文本注入
   - 设置保存与回滚
   - overlay 显示与关闭

### 4.2 扩大发布
在灰度验证无异常后，逐步扩大发布范围。

## 5. 发布后观察项

重点观察：

- worker 是否异常退出
- 热键是否失效
- overlay 是否不显示或卡住
- 注入是否失败
- 配置保存后是否出现运行时不一致
- 日志中是否出现新型异常栈

## 6. 回滚触发条件

出现以下任一情况建议立即回滚：

1. 核心业务路径不可用
   - 无法启动识别
   - 无法完成最终提交
2. 大面积兼容问题
   - 热键失效
   - 注入全面失败
   - overlay 异常导致不可用
3. worker 生命周期异常频发
4. 设置保存后导致持续性运行时错误

## 7. 回滚步骤

1. 停止当前 agent 进程
2. 恢复上一版本代码/构建产物
3. 恢复 `%APPDATA%/DoubaoVoiceInput/` 备份
4. 重启 agent
5. 执行最小冒烟验证：
   - 热键
   - 识别启动
   - 最终结果
   - overlay
   - 设置保存
